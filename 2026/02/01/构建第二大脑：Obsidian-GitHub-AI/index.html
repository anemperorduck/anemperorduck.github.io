构建第二大脑：Obsidian+GitHub+AI

发表于2026-02-01|更新于2026-02-02|教程
|浏览量:

这几天自己部署了个Gemini CLI，刚开始用就发现Gemini CLI+ Obsidian兼职是天作之合，我相信这对我们日常构建自己的“知识库”而言是非常有价值的技术栈组合。Obsidian(本地知识库) + GitHub (版本控制/同步) + Gemini CLI (AI 生产力)，这三者结合构建了一个本地化的、免费的，又具备无限扩展性和智能化能力的“第二大脑”。

Obsidian 作为新生代的 Markdown 笔记软件，拥有极低的内存占用和媲美 VSCode 的插件生态；
GitHub 完美解决了 Obsidian 的远程同步与版本回溯问题，让数据真正掌握在自己手中；
Markdown 的纯文本特性天生具有强大的 AI 亲和力，配合 Gemini CLI 等 AI Agent，可以让我们在本地直接对笔记进行深度处理，如虎添翼。

下面将会详细介绍如何搭建这套 Obsidian + GitHub + Gemini CLI 的全能工作流。
同时也会附加一些在我自己使用过程对 obsidian 进行的优化。

一、 简单介绍

1. Obsidian：坚实的基石

  • 本地存储：数据也是资产，保存在本地硬盘比云端更安全。
  • 双向链接：构建知识图谱，让笔记之间产生联系。
  • 插件生态：从看板到日历，功能无限扩展。

2. GitHub：免费的云端同步和版本控制

  • 全平台同步：利用 Git 协议实现 PC、Mac、移动端的数据互通。
  • 版本控制：Git 的回滚就是世界上最好的后悔药。

3. AI 增强 (Gemini CLI)：第二大脑

不同于 Obsidian 内部的 AI 插件，使用 CLI 版本的 AI Agent 工具处理本地 Markdown 文件有独特优势：

  • 模型质量：高质量模型—— Gemini Pro/Flash 。
  • 批处理能力：可以让ai一次性整理数百篇笔记、或者发挥自己的想象力让ai完成各种各样的任务。
  • 零成本：Google AI Studio 目前对个人开发者提供较为慷慨的免费额度，几乎免费。

2. 安全预防（防冲突指南）

Git 的本质是版本控制，而非类似于 iCloud 的实时增量同步。

  • 重点：避免手机和电脑同时编辑同一个文件。
  • 操作习惯：
    • 开始工作前：先 Pull（拉取最新变动）。
    • 结束工作后：等待 Push 完成（或手动 Ctrl+P 输入 Git: Push）。
  • 冲突解决：如果不慎发生冲突，Git 会做出提示，通常手动保留需要的版本即可，或者在使用 git merge合并。

三、 Obsidian 图片存储优化

Obsidian 原生的图片管理非常潦草（全堆在根目录），且obsidian使用的链接的格式并不通用，所以我们需要通过配置实现“Typora 式”的原生md图片管理方式来优化我们的使用体验。

1. 原生方案的缺陷

  • 文件混乱：图片与笔记混杂，难以管理。
  • 兼容性差：![[image.png]] 是 Wiki 链接，离开 Obsidian 后（如在 VSCode、GitHub、博客）无法正常显示。

2. 解决方案

第一步：规范化设置

  1. 进入 Obsidian 设置 -> 文件与链接。
  2. 取消勾选“使用 Wiki 链接”（这样会生成标准的 ![](url) Markdown 语法）。
  3. 内部链接类型：选择“基于当前笔记的相对路径”。

第二步：安装 Custom Attachment Location

  1. 安装社区插件 Custom Attachment Location。
  2. 进入插件设置，进行如下配置：
    • Location for New Attachments: In subfolder under current folder (在当前文件夹下的子文件夹)
    • Subfolder Name: assets/${filename} (这将为每篇笔记创建一个同名的资源文件夹，方便管理)
    • Renaming: 选择 Auto-rename attachment folder (重命名笔记时，图片文件夹自动跟随改名)

3. 最终效果

当你在一篇名为 MyNote.md 的笔记中粘贴图片时：

  1. 图片会自动存入 assets/MyNote/ 文件夹。
  2. Markdown 链接变为 ![](assets/MyNote/image-2023.png)。
  3. 全平台兼容：无论你在 GitHub 网页预览，还是用 VSCode 打开，图片均能完美显示。
  4. 尺寸调整：依然支持 ![text|300](path.png) 的语法来调整大小。

四、 AI 增强：Gemini CLI 实战

为什么不直接用Obsidian的插件市场提供了丰富的AI插件？很多AI插件非常好用，但是他们功能往往没有AI Agent这么强大，与其把AI锁死在聊天框，我更希望他能充分介入我的生活，成为我生活中更加强有力的工具。

我们选择谷歌官方推出的Gemini CLI，他深度集成了Google的免费Tier（每分钟 60 次请求，每天 1000 次），这对于个人开发者而言几乎是无限的。

五、 移动端 (PE) 配置

手机端主要用于查阅和灵感捕捉，复杂的编辑建议回到电脑端。

  1. 仓库克隆：
    • 将笔记文件夹复制至手机 Documents/
  2. 认证配置：
    • 进入 Obsidian Git 插件设置。
    • Authentication User: 填写 GitHub 用户名。
    • Authentication Password: 填写 GitHub PAT (Personal Access Token)。
  3. 操作流：
    • 打开手机 Obsidian -> 下拉菜单 -> Pull（拉取更新）。
    • 修改内容。
    • 下拉菜单 -> Push（推送到云端）。

六、 导出与分享

当笔记通过 Markdown 标准化和图片相对路径化后，导出变得异常简单。

  • 插件：Enhancing Export。
  • 依赖工具：Pandoc (预先安装)。
  • 优势：
    • 支持一键导出为 Word (docx)、PDF、HTML。
    • 得益于之前的图片优化配置，Pandoc 能准确找到相对路径下的图片并将其嵌入到 Word 或 PDF 中，不会出现图片丢失的尴尬。
