从零搭建国内API定向的Claude Code实践指南

发表于2026-03-19|更新于2026-03-19|教程
|浏览量:

一、为什么要用 Claude Code？

Claude Code 是 Anthropic 官方推出的命令行编程助手，与传统的 AI 编程工具有本质区别：

对比维度 ChatGPT / Claude Web IDE 插件 (如 Copilot) Claude Code
交互方式 网页对话 行内补全/聊天窗口 命令行交互
上下文范围 手动粘贴代码片段 当前文件或有限上下文 整个项目目录
操作能力 仅文本输出 代码补全、有限重构 可执行命令、修改文件、运行测试

Claude Code 的核心优势在于项目级上下文理解。它能够：

  • 自动读取项目中的多个文件，理解代码结构
  • 直接编辑代码文件，而非仅提供建议
  • 执行 shell 命令，运行测试、构建项目
  • 与 git 深度集成，处理分支、提交、PR 等操作

这种 CLI + 项目级上下文的组合，使其成为真正意义上的”AI 结对编程伙伴”，而不仅仅是代码补全工具。

原因分析：

安装脚本会从 storage.googleapis.com 下载 Claude Code 的二进制文件。即使配置了 VPN，Google Cloud Storage 服务在某些网络环境下仍然会被阻断。这是因为：

  1. VPN 可能仅代理 HTTP/HTTPS 流量，而脚本可能尝试其他协议
  2. 部分企业网络或 ISP 对 Google Cloud 服务有特殊限制
  3. DNS 解析可能被污染，导致无法正确访问目标服务器

hasCompletedOnboarding 的作用：

5.1 原理：API 重定向

Claude Code 默认连接 Anthropic 官方 API (api.anthropic.com)。在国内网络环境下，直接访问存在困难。

为什么可以”伪装 Claude”？

Anthropic API 已成为事实上的标准，多家国内厂商提供了兼容接口：

  • 请求格式兼容（Headers、Body 结构）
  • 响应格式兼容（JSON 结构、SSE 流式输出）
  • 模型能力映射（将 Claude 模型名映射到国产模型）

这意味着 Claude Code 无需修改任何代码，只需修改请求目标地址，就能使用国产模型。

5.3 参数逐项详解

ANTHROPIC_BASE_URL

作用： 指定 API 请求的目标地址。

本质： 这是智谱 GLM 提供的 Anthropic 兼容接口。Claude Code 所有 API 请求都会发送到此地址，而非 Anthropic 官方服务器。

踩坑提示： 确保地址末尾没有多余的斜杠，某些 API 网关对路径匹配非常严格。

ANTHROPIC_AUTH_TOKEN

作用： API 认证密钥。

本质： 这是你在智谱开放平台申请的 API Key。Claude Code 会将其放入请求头 x-api-key 中发送。

获取方式： 访问 智谱开放平台 注册账号并创建 API Key。

踩坑提示： 不要将真实的 API Key 提交到公开仓库。建议使用环境变量或配置文件忽略机制保护密钥。

ANTHROPIC_MODEL

作用： 指定默认使用的模型。

本质： Claude Code 会请求名为 glm-5 的模型。由于使用了 GLM 的兼容接口，这个模型名会被智谱后端识别为其 GLM-5 模型。

ANTHROPIC_DEFAULT_HAIKU_MODEL / SONNET_MODEL / OPUS_MODEL

作用： 模型映射配置。

本质解释：

Claude Code 内部使用三档模型分级：

Claude 模型档位 原始定位 映射到 GLM
Haiku 轻量快速 glm-4.7
Sonnet 平衡型 glm-5
Opus 旗舰级 glm-5-turbo

这些配置告诉 Claude Code：当你想要调用 Haiku 级别模型时，实际发送 glm-4.7；当需要 Sonnet 级别时，发送 glm-5；当需要使用Opus时，发送glm-5-turbo（glm-5 针对 Agent 长链路任务的优化模型）

为什么这样映射？

  • GLM 系列的模型分级与 Claude 不完全对应
  • 根据实际性能和成本，选择最适合的映射方案
  • 用户可根据自己的需求调整映射关系

API_TIMEOUT_MS

作用： API 请求超时时间（毫秒）。

本质： 设置为 300000ms（5 分钟），给复杂任务留足响应时间。

踩坑提示： 如果频繁遇到超时错误，可适当增大此值。但过大的超时可能导致长时间无响应，影响使用体验。

CLAUDE_CODE_DISABLE_NONESSENTIAL_TRAFFIC

作用： 禁用非必要网络请求。

本质： Claude Code 默认会发送一些遥测数据、版本检查等请求。设置为 1（启用禁用）可以：

  • 减少不必要的网络请求
  • 加快启动速度
  • 避免遥测请求失败导致的错误提示

踩坑提示： 这是提升国内使用体验的关键配置。禁用后，Claude Code 几乎所有网络请求都只会发往你配置的 ANTHROPIC_BASE_URL。

六、总结

本方案的本质可以用一句话概括：

通过环境变量重定向 API 请求，让 Claude Code 使用兼容 Anthropic 接口的国内模型服务，实现国内网络环境下的稳定使用。

核心步骤：

  1. 使用 winget install Anthropic.ClaudeCode 安装
  2. 配置 claude.json 跳过新手引导
  3. 在 settings.json 中设置 API 重定向和模型映射
  4. 禁用非必要的网络请求

这套方案的优势在于无需修改 Claude Code 源码，完全通过官方支持的配置方式实现，便于后续版本升级和维护。
